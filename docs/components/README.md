## Maintenance

- Add new component docs inside the closest numbered section, using kebab-case filenames.
- Keep the file basename normalized to the exported component name so `npm run audit:components` can count direct docs coverage.
- If a component does not fit an existing section, add a new numbered section instead of placing the page at `docs/components/*.md`.
- Re-run `npm run audit:components` after adding, moving, or renaming component docs.
