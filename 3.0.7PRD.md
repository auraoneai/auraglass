## Problem Statement

The current redesigned homepage has a trust gap:

- It looks premium.
- It labels visuals as `<GlassButton />`, `<GlassInput />`, `<GlassTabs />`, `<GlassToast />`, etc.
- But the visible UI is often custom CSS and plain HTML elements, not real package components.

This means the homepage is visually persuasive but technically misleading. A user may reasonably assume AuraGlass can create that exact interface out of the box, while the package currently lacks several homepage-specific primitives.

The fix is not to delete the redesign. The fix is to upstream the design language into AuraGlass so the homepage can honestly use the package.

## Accessibility Requirements

All marketing kit components must meet the same accessibility bar as app components.

- Background/orb/logo visuals are decorative by default and must not pollute screen reader output.
- `LogoMark` supports an accessible label when used as branding.
- `DisplayText` renders semantic headings using `as`.
- Gradient text must have high-contrast fallback.
- Buttons keep visible focus.
- Copy interactions announce copied state.
- Reduced-motion disables drifting/pulsing/particle animations.
- No content depends only on animation.
- No rapid flashing.

## SSR And Hydration Requirements

The homepage currently relies on client-side motion. The package components must be safe in React/Next environments.

- No direct `window`, `document`, or `navigator` access during render.
- Clipboard access only inside guarded event handlers.
- Particle positions use seeded deterministic generation.
- Random IDs use stable deterministic IDs or React `useId`.
- Motion values should initialize consistently server/client.
- Components should work in Next App Router.

## Performance Requirements

The marketing kit must look premium without introducing avoidable GPU or memory issues.

- `AuroraBackground` should use CSS gradients and transforms, not per-frame canvas by default.
- Particle count defaults must be conservative.
- `motion="full"` can be opt-in; default should be performant.
- Reduced-motion and low-power modes should reduce animation work.
- No `requestAnimationFrame` loops unless required and cleaned up.
- No `gl.readPixels()`-style per-frame stalls.
- Components should not significantly increase base bundle for users who do not import marketing kit components.
